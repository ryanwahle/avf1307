Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

Having one login to multiple websites and payment systems are a real benefit for users. No longer do they have to remember multiple passwords, they can just use their Twitter account to login to everything. This also includes payments, where instead of having an actual PayPal account, PayPal would link to the users Twitter account. It’s really great for the user.

Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

After going to programmableweb.com, I could only find a data source listed for JSON, not JSON-P. The search results are so large, that it’s clear that JSON is a create format to transfer data between different applications, really only for the popularity. And it’s only popular because it fits the needs of the developers.

I searched the web for what the difference is, and we need JSON-P because with JSON requests, you can not request data from a website that is not on the same server as your own application. Since we are trying to retrieve data from sources that other company provide, we need JSON-P.
